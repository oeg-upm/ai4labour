Description of the terminology
In this section, we are going to explain the way we have done the terminology about crimes. 

Data

First of all, we use a CSV file in which we collect each crime term, with its information, which includes an URI, synonyms, relations between others terms, definition from the legal dictionary and in some cases, some extra information.  

After cleaning all the data, we are ready to go further in the process. The RMLMapper helps us to convert the CSV file into RDF, using customized mappings that are coded in RML.

Skos (Simple Knowledge Organization System)

We use the Skos' vocabulary because of its ability to represent and organize taxonomies.

The relationships we use from this vocabulary are:

  • skos:Concept → define the crime term we are going to modelized.
  • skos:prefLabel → define how the term is known.
  • skos:altLabel → define synonyms 
  • skos:narrower → enable the representation of hierarchical links between terms. It represents more specific terms.
  • skos:broader → enable the representation of hierarchical links between terms. It represents more general terms.
  • skos:note → add extra documentation about the term.
  • skos:definition → add, if exists, the definition in the Diccionario panhispánico del español jurídico.

 

The hierarchical links we have built are based on the internal code of each term. For example, the term A001000000 is broader than the term A001002000, A001003000, A001004000, A001005000, A001006000, A001007000 and each of those terms are narrower than A001000000.
