Overview

easySKOS was born as the development part of the final undergraduate project in Computer Engineering at the Polytechnique University of Madrid. It is a free web application that transforms your data in CSV or XLSX spreadsheets into SKOS-based files, where its main objective is to reduce entry barriers for people who do not know about the linked data domain.
The current version works statically, which means that every template has its own mapping rules and that rules run inside the RMLmapper engine. Therefore, the main restraint is to neither change the name of the template nor its columns' structure, because the mapper could not link the mapping rules to the source file.

Templates

easySKOS provides three ready-to-use templates (in CSV or XLSX format), each one covering a popular SKOS-based usage. Just fill the cells with your data and easySKOS will do the rest. Moreover, all the columns are optional except the ones marked with * have to be fulfilled because easySKOS uses them to create the International Resource Identifier (IRI) of the term.
The IRI is constructed following the next pattern:

<http://terminoteca.linkeddata.es/kos/TEMPLATE/BASE IRI/SCHEME/ID>

However, it is important to highlight the importance of not changing the name of the template and its structure, because the transformation could not be done.
To sum up:

  • Download the template provided in the converter page.
  • Fill the the template with your data. If you do not have information for a column, just let it blank, do not remove the column.
  • Upload the template with the original filename and wait until easySKOS finishes.
EUROVOC

Template for the multilingual and multidisciplinary thesaurus of the EU. Each concept is defined by six columns to set up its metadata, columns for the preferred and alternative labels (in twenty-seven languages), and three more columns to establish its hierarchy.
The names of the columns are:

  • Metadata definition. Note that special characters are not allowed in these columns:
    • ID*: Identifier that is used to form the International Resource Identifier (IRI) of each term.
    • BASE IRI*: Represents the exact path to access the resource. No special characters are allowed.
    • SCHEME*: Additional information used to classify the terms.
    • CREATED dd-MM-yyyy: Date of creation.
    • VERSION: Actual version of the term.
    • STATUS: Actual status of the term.
  • Concept definition. Note that the quotes can cause problems if they are not closed.
    • PREFERRRED LABEL: Describe the concept in a clear and unambiguos way. There are 27 columns, each one for each european languages.
    • ALTERNATIVE LABEL: List synonyms, acronyms, or abbreviations available for a concept when necessary. There are 27 columns, each one for each european languages.
    • BROADER CONCEPT: Represents, in English, that this concept has broader concept, which means that this concept is the more specific one. NOTE: a SKOS concept can be attached to several broader concepts at the same time.
    • NARROWER CONCEPT: Represents, in English, that this concept has narrower concept, which means that this concept is the more generic one.
    • RELATED: Used to assert an associative link between two SKOS concepts.
Example of Broader/Narrower Relationships 
    ex:animals rdf:type skos:Concept;
      skos:prefLabel "Animals"@en;
      skos:narrower ex:mammals.
                          
    ex:mammals rdf:type skos:Concept;
      skos:prefLabel "Mammals"@en;
      skos:broader ex:animals.
skos:broader and skos:narrower are NOT declared as transitive properties. They are only used to assert a direct hierarchical link between two SKOS concepts.
If you have more than one alternative label, broader or narrower concept, you have to duplicate the information in another row.
Glossary

Template for a common glossary, which includes three columns to set up its metadata, eight columns to name and define each term, in four languages (English, Spanish, French and Italian).
The names of the columns are:

  • Metadata definition. Note that special characters are not allowed in these columns.
    • ID*: Identifier that is used to form the International Resource Identifier (IRI) of each term.
    • BASE IRI*: Represents the exact path to access the resource. No special characters are allowed.
    • SCHEME*: Additional information used to classify the terms.
  • Concept definition. Note that the quotes can cause problems if they are not closed.
    • ES: Term in Spanish.
    • EN: Term in English.
    • FR: Term in french.
    • IT: Term in italian.
    • DEFINICION: Definition in Spanish.
    • DEFINITION: Definition in English.

    • DEFINIZIONE: Definition in italian.
Hierarchical taxonomy

Template for a hierarchical taxonomy, which includes three columns to set up its metadata and four columns to establish the preferred, alternative label, and broader or narrower concept. It also includes four properties to define each term in three languages (English, Spanish, and Italian) and to write notes related to the term.
The names of the columns are:

  • Metadata definition. Note that special characters are not allowed in these columns:
    • ID*: Identifier that is used to form the International Resource Identifier (IRI) of each term.
    • BASE IRI*: Represents the exact path to access the resource. No special characters are allowed.
    • SCHEME*: Additional information used to classify the terms.
  • Concept definition. Note that the quotes can cause problems if they are not closed.
    • PREFERRRED LABEL: Describe the concept in a clear and unambiguos way.
    • ALTERNATIVE LABEL: List synonyms, acronyms, or abbreviations available for a concept when necessary.
    • BROADER CONCEPT: Represents, in English, that this concept has broader concept, which means that this concept is the more specific one. NOTE: a SKOS concept can be attached to several broader concepts at the same time.
    • NARROWER CONCEPT: Represents, in English, that this concept has narrower concept, which means that this concept is the more generic one.
    • DEFINICION: Definition in Spanish.
    • DEFINITION: Definition in English.
    • DEFINIZIONE: Definition in italian.
    • NOTE: Additional information.
Remember, if you have more than one alternative label, broader or narrower concept, you have to duplicate the information in another row.

Common errors

There basically two errors that may appear when using the easySKOS converter.

  • An error occurred when transforming your template: It means that some columns of the template have been changed or the RMLmapper engine is having problems.
    • Check whether the colums name were changed and if not, please let us know the problem.
    • Check whether quotes have been added and remove them. If not, please let us know the problem.
  • Cannot set the mapping rules for that file: It means that the filename of the template was changed and the converter is not able to link the mapping rules with source file.
    • Check whether the filename was changed and if not, please let us know the problem.
